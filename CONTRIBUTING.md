# Guide de Contribution à AfriTest

Merci de l'intérêt que vous portez à AfriTest ! Nous accueillons avec plaisir toutes les contributions qui aident à améliorer la plateforme pour les développeurs et testeurs africains.

## Comment contribuer ?

### 1. Rapporter des Bugs
Si vous trouvez un bug, veuillez ouvrir une "Issue" sur GitHub en utilisant le modèle de rapport de bug. Soyez aussi précis que possible sur les étapes pour reproduire le problème.

### 2. Suggérer des Améliorations
Les suggestions de fonctionnalités sont les bienvenues ! Utilisez le modèle de "Feature Request" pour nous faire part de vos idées.

### 3. Soumettre du Code
- Forkez le dépôt.
- Créez une branche descriptive (ex: `feature/nom-de-la-feature` ou `fix/nom-du-bug`).
- Assurez-vous que votre code respecte les standards PSR-12 pour PHP.
- Testez vos changements localement.
- Soumettez une Pull Request vers la branche `main`.

## Standards de Code

- **PHP** : Suivez les standards Laravel et PSR-12.
- **JS/React** : Utilisez des composants fonctionnels et suivez les conventions de nommage CamelCase.
- **CSS** : Utilisez Tailwind CSS pour tout le style.

## Processus de Pull Request

1. Assurez-vous que la PR décrit clairement le changement.
2. Si la PR corrige un bug, liez-la à l'Issue correspondante (`fixes #issue_number`).
3. Un mainteneur passera en revue votre code et pourra demander des modifications.

## Code de Conduite
En participant à ce projet, vous acceptez de respecter un comportement professionnel et respectueux envers tous les membres de la communauté.
